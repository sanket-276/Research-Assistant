# Status Filter Removal & Question-Based Search with Citations

## Overview
Removed the status filter dropdown from the dashboard and implemented intelligent question-based search that shows answers with proper citations and sources when users ask questions.

## Changes Made

### 1. **Removed Status Filter Dropdown** ❌

**Before**: Dashboard had a status filter dropdown with options:
- All Status
- Completed
- Processing  
- Failed

**After**: Status filter completely removed from the UI

**Reason**: Simplified interface - users don't need to filter by processing status since all documents are searchable regardless of status.

**Code Removed**:
```javascript
<select value={filterStatus} onChange={(e) => setFilterStatus(e.target.value)}>
  <option value="all">All Status</option>
  <option value="completed">Completed</option>
  <option value="processing">Processing</option>
  <option value="failed">Failed</option>
</select>
```

**State Cleanup**:
- Removed `filterStatus` state variable
- Removed status filtering logic from `getFilteredAndSortedDocs()`
- Simplified document filtering code

---

### 2. **Intelligent Question-Based Search** 🤖

**New Feature**: When users search using question patterns, the system now:
1. Detects if the search is a question
2. Retrieves relevant passages from documents
3. Generates a concise answer using AI
4. Shows citations and sources

**Question Detection**:
Searches are treated as questions if they:
- Start with question words: `what`, `how`, `why`, `when`, `where`, `who`, `which`, `can`, `does`, `is`, `are`, `explain`, `describe`, `tell`
- End with a question mark `?`

**Examples**:
- "What is machine learning?" → Question-based search
- "How does climate change work?" → Question-based search
- "Explain neural networks" → Question-based search
- "machine learning" → Regular keyword search

### 4. **Backend Implementation** 🔧

**New Endpoint**: `/api/chat/answer/`

**Process**:
1. Receives question from frontend
2. Retrieves user's accessible documents
3. Uses RAG (Retrieval Augmented Generation) to find top 5 relevant passages
4. Combines passages into context
5. Sends to Groq LLM with prompt
6. Returns structured response with:
   - Answer text
   - Source citations (filename, page, quote, URL)
   - Related document IDs

**Code Structure**:

## Technical Details

### Frontend State:
```javascript
const [searchAnswer, setSearchAnswer] = useState(null);
const [searchSources, setSearchSources] = useState([]);
```

### API Call:
```javascript
const response = await fetch('/api/chat/answer/', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Token ${localStorage.getItem('authToken')}`
  },
  body: JSON.stringify({ question: searchQuery })
});

const data = await response.json();
setSearchAnswer(data.answer);
setSearchSources(data.sources);
```

### Backend Libraries:
- **RAG**: Uses `combined_retrieval()` from `services.py`
- **LLM**: Groq API with `llama-3.1-8b-instant` model
- **Vector DB**: ChromaDB for semantic search
- **TF-IDF**: Custom implementation for keyword matching
